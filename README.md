## 📋 О проекте

**Incoming Mail Register** — это полнофункциональная веб-платформа для учета, отслеживания и управления входящей корреспонденцией. Система разработана для организаций, которым необходимо централизованное хранение писем, контроль сроков исполнения, автоматические уведомления и детальная отчетность.

### 🎯 Основные возможности

| Функция | Описание |
|---------|----------|
| 📝 **Управление письмами** | Полный CRUD для входящих документов с регистрацией номера, даты, отправителя, содержания и срока исполнения |
| 👥 **Пользователи и роли** | Регистрация, авторизация (JWT), разделение на обычных пользователей и администраторов |
| 🏢 **Отделы** | Поддержка трех отделов: Школьный отдел, Отдел ГИА, Департмент |
| 🔄 **Передача писем** | Возможность передавать письма между сотрудниками внутри отдела (админ может передавать между любыми отделами) |
| ⏰ **Уведомления** | Автоматические напоминания за 5 дней до срока исполнения (Email, SMS, браузерные) |
| 📁 **Журналы потоков** | Автоматическая группировка писем по потокам (все, что после `/` в номере письма) |
| 📋 **Письма-запросы** | Специальная маркировка писем для отчетности (Для собственных целей / Для Минпросвещения России) |
| 📊 **Дашборд** | Визуальная статистика по письмам, статусам и срокам |
| 🔐 **Админ-панель** | Управление пользователями (блокировка, удаление, назначение администраторов) |
| 📎 **Экспорт в Excel** | Экспорт списка писем, писем потока и писем-запросов в формат Excel |
| 🎨 **Темная/светлая тема** | Адаптивный интерфейс с поддержкой двух тем |
| 📱 **Адаптивный дизайн** | Корректное отображение на всех устройствах |

## 🚀 Технологии

### Backend
- **[FastAPI](https://fastapi.tiangolo.com/)** — современный веб-фреймворк для Python
- **[SQLAlchemy](https://www.sqlalchemy.org/)** — ORM для работы с базой данных
- **[SQLite](https://sqlite.org/)** / PostgreSQL — легковесная БД для разработки и продакшена
- **[JWT](https://jwt.io/)** — аутентификация и авторизация
- **[bcrypt](https://pypi.org/project/bcrypt/)** — хеширование паролей
- **[APScheduler](https://apscheduler.readthedocs.io/)** — планировщик уведомлений

### Frontend
- **[Jinja2](https://jinja.palletsprojects.com/)** — шаблонизатор
- **CSS Custom Properties** — поддержка темной/светлой темы
- **Адаптивная верстка** — медиа-запросы для мобильных устройств

### Дополнительно
- **[OpenPyXL](https://openpyxl.readthedocs.io/)** — экспорт в Excel
- **[Gunicorn](https://gunicorn.org/)** + **Nginx** — продакшен-деплой

## 🛠 Установка и запуск

### Требования
- Python 3.10+
- pip

### Локальная установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/your-username/incoming_correspondence.git
cd incoming_correspondence

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Создайте файл .env
cat > .env << EOF
SECRET_KEY=your-secret-key-here
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
EOF

# 5. Запустите приложение
python run.py
```

Откройте браузер: **http://127.0.0.1:8000**

**Учетные данные по умолчанию:**
- Логин: `admin`
- Пароль: `admin123`

## 🐳 Деплой на сервер

### Для сервера Ubuntu (с Nginx и Supervisor)

```bash
# 1. Копируем проект на сервер
scp -r incoming_correspondence root@your-server:/var/www/incoming/

# 2. Устанавливаем зависимости
cd /var/www/incoming/incoming_correspondence
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install gunicorn

# 3. Настраиваем Supervisor
nano /etc/supervisor/conf.d/incoming.conf

# 4. Настраиваем Nginx
nano /etc/nginx/sites-available/incoming

# 5. Устанавливаем SSL (Let's Encrypt)
certbot --nginx -d your-domain.ru

# 6. Запускаем сервисы
supervisorctl reread && supervisorctl update
systemctl restart nginx
```

Подробная инструкция деплоя доступна в [DEPLOY.md](DEPLOY.md).

## 📊 Функционал в деталях

### 📝 Письма

- **Создание письма**: входящий номер, даты, отправитель, содержание, срок исполнения
- **Статусы**: В ожидании, В работе, Исполнено, Просрочено, Передано
- **Исполнитель**: назначение ответственного сотрудника
- **Передача**: возможность передать письмо другому сотруднику

### 📁 Журналы потоков

Поток извлекается автоматически из входящего номера — всё, что после символа `/`.

Примеры:
- `642/07-25` → поток `07-25`
- `123/ДСП-01` → поток `ДСП-01`

### 📋 Письма-запросы

Для писем, требующих отчетности, можно указать:
- **Тип запроса**: Для собственных целей / Для Министерства просвещения России
- **Примечание**: дополнительная информация для отчетности

### 🔔 Уведомления

- Автоматическая проверка писем каждые 6 часов и в 9:00
- Уведомление за 5 дней до срока исполнения
- Каналы: Email, SMS (API), браузерные уведомления
